## 🧠 개발 철학
### 1) 데이터로 판단한다.  
감이 아니라 **수치·로그·스택트레이스**로 문제를 정의하고 해결한다.

### 2) 문제는 숨기지 않는다.  
에러는 감추지 않고 **노출·분류·문서화**해야 해결 속도가 올라간다.

### 3) 테스트는 안정성의 기준이다.  
테스트는 문제의 재발을 막는 장치이며, 구조적 안정성을 확보하는 핵심이다.

### 4) 기술 선택에는 이유가 있어야 한다.  
성능, 안정성, 유지보수성을 기준으로 **선택 → 실험 → 적용**의 흐름을 따른다.
